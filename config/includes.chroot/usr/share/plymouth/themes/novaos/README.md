  /usr/share/plymouth/themes/novaos/

## How Plymouth theme files get here

Plymouth theme files are generated by scripts and installed by the build hook.
They are NOT committed here directly (they live in novaos-config/).

## Build-time flow

  novaos-config/plymouth-theme/novaos/     ← source of truth (all files present)
  ├── novaos.plymouth                       ← theme descriptor
  ├── novaos.script                         ← Plymouth script theme
  ├── logo.png                              ← rendered from novaos-logo.svg
  ├── progress_box.png                      ← generated by render-assets.sh
  ├── progress_box_thumb.png
  ├── lock.png / unlock.png / spinner.png
       │
       ▼  (scripts/install-branding.sh → copies to /usr/share/plymouth/themes/novaos/)
  /usr/share/plymouth/themes/novaos/        ← in chroot (via install-branding.sh)
       │
       ▼  (config/hooks/live/005-novaos-branding.hook.chroot)
  plymouth-set-default-theme novaos         ← selected as default in ISO

## For Person 1 (build engineer)

1. Run render-assets.sh on a Linux machine first (generates logo.png, etc.)
2. Run install-branding.sh inside the chroot (or add .deb to package-lists)
3. lb build picks up 005-novaos-branding.hook.chroot automatically
4. The hook runs plymouth-set-default-theme + update-initramfs

## novaos.plymouth format reference

  [Plymouth Theme]
  Name=NovaOS
  Description=NovaOS boot splash
  ModuleName=script

  [script]
  ImageDir=/usr/share/plymouth/themes/novaos
  ScriptFile=/usr/share/plymouth/themes/novaos/novaos.script
